Insert operations CRUD — 17%+
// insertOne — returns { acknowledged, insertedId } await col.insertOne({ title: "Dune", year: 2021 }) // insertMany — returns { acknowledged, insertedCount, insertedIds } await col.insertMany([{ title: "A" }, { title: "B" }]) // ordered: true (DEFAULT) — stops on first error // ordered: false — skips errors, continues inserting
⚠ Exam trap — ordered inserts:
ordered: true (default) → stops at first error, earlier docs stay inserted
ordered: false → skips bad docs, inserts all valid ones
Find operations + cursor CRUD
findOne()
Returns a document or null.
Awaitable directly.
find()
Returns a Cursor.
Must iterate with toArray() or for await.
// findOne → document or null const doc = await col.findOne({ year: 2021 }) // find → cursor (NOT documents yet) const cursor = col.find({ year: 2021 }) const all = await cursor.toArray() // loads all into memory for await (const d of col.find({ year: 2021 })) { } // memory efficient // Chaining on cursor col.find({}).sort({ year: -1 }).limit(5).skip(10)
⚠ Exam trap: find() returns a Cursor, not an array or Promise of documents. Calling await col.find() gives you the cursor object, not the data.

Array queries — the #1 exam trap High frequency
// 4 DIFFERENT queries on { genres: ["Action","Drama","Comedy"] } { genres: "Action" } // ✅ matches — "Action" is anywhere in the array { genres: ["Action", "Drama"] } // ❌ does NOT match — requires EXACT array ["Action","Drama"] only { genres: { $all: ["Action", "Drama"] } } // ✅ matches — contains BOTH, any order, other elements OK { genres: { $in: ["Action", "Horror"] } } // ✅ matches — contains Action OR Horror (or both) { genres: { $size: 3 } } // ✅ matches — array has exactly 3 elements
PatternMeans
{ arr: "X" }Array contains "X"
{ arr: ["X","Y"] }Array is EXACTLY ["X","Y"] in that order
{ arr: { $all: ["X","Y"] } }Array contains X AND Y (any order)
{ arr: { $in: ["X","Y"] } }Array contains X OR Y
{ arr: { $size: N } }Array has exactly N elements
Projections — include / exclude fields Know the exception
{ title: 1, year: 1 } // ✅ include title, year (+ _id by default) { title: 1, year: 1, _id: 0 } // ✅ include title, year, exclude _id { plot: 0, runtime: 0 } // ✅ exclude plot, runtime (return everything else) { title: 1, plot: 0 } // ❌ ERROR — cannot mix 1 and 0 (except _id)
⚠ Rule: Cannot mix inclusion (1) and exclusion (0) in the same projection — except _id: 0 which can appear with any inclusion projection. This is the one and only exception.
Dot notation — querying nested fields Used everywhere
// Document: { imdb: { rating: 8.8, votes: 2300000 } } // Always use quotes around dot notation keys col.find({ "imdb.rating": { $gte: 8.0 } }) // ✅ correct col.find({ imdb.rating: { $gte: 8.0 } }) // ❌ syntax error (no quotes)
Must-remember rules for exam day Quick-fire
⭐ _id is immutable — cannot be updated ever
⭐ find() returns a Cursor, not documents
⭐ Use Decimal128 for money / precise decimals — never Double
⭐ Array exact match: { arr: ["A","B"] } — order matters, nothing else allowed
⭐ Cannot mix inclusion + exclusion in projection (except _id: 0)
⭐ insertMany default is ordered: true — stops on first error
⭐ Documents in same collection can have different shapes — schema-flexible
⭐ Max document size = 16 MB
⭐ Dot notation keys must always be in quotes in queries
